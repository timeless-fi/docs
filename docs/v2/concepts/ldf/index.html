Liquidity Density Functions (LDFs)

Liquidity Density Functions (LDFs) are a core concept in Bunni v2 that enable efficient liquidity distribution, modification, and swaps with constant gas costs. LDFs provide a new language for specifying liquidity distributions that builds directly on top of Uniswap v3's concept of ticks.

What are LDFs?​

An LDF is a normalized function that defines how liquidity is distributed over "ricks" (rounded ticks) in a Bunni v2 pool. Mathematically, an LDF is a function LDFw:R→[0,1]LDF_w: R \rightarrow [0, 1]LDFw​:R→[0,1], where RRR is the rick space and www is the tick spacing.

Given a pool with total liquidity LLL and liquidity lrl_rlr​ at rick rrr, we can compute lrl_rlr​ using the LDF:

lr=L⋅LDFw(r)l_r = L \cdot LDF_w(r)lr​=L⋅LDFw​(r)

LDFs are normalized, meaning that ∑rLDFw(r)=1\sum_r LDF_w(r) = 1∑r​LDFw​(r)=1.

Rick Indices​

To simplify the representation of LDFs, we can use rick indices. Given tick spacing www, rick rrr, and origin μ\muμ, the rick index xxx is defined as:

x=r−μwx = \frac{r - \mu}{w}x=wr−μ​

This allows us to represent an LDF alternatively as:

d(x)=LDFw(wx+μ)d(x) = LDF_w(wx + \mu)d(x)=LDFw​(wx+μ)

LDFw(r)=d(r−μw)LDF_w(r) = d(\frac{r - \mu}{w})LDFw​(r)=d(wr−μ​)

dα,l(x)={αx(1−α)1−αlx∈[0,l)∩Z0otherwised_{\alpha,l}(x) = \begin{cases} \frac{\alpha^x(1-\alpha)}{1-\alpha^l} & x \in [0, l) \cap \mathbb{Z} \\ 0 & \text{otherwise} \end{cases}dα,l​(x)={1−αlαx(1−α)​0​x∈[0,l)∩Zotherwise​

This distribution allocates liquidity over lll ricks following a geometric pattern. By adjusting the origin μ\muμ, the LDF can be shifted over the rick space. By modifying α\alphaα and lll, the shape of the distribution can be changed.

Cumulative Amount Functions (CAFs)​

Each LDF has two associated Cumulative Amount Functions (CAFs), A0:R∪rmax+w→R>0A_0: R \cup r_{max} + w \rightarrow \mathbb{R}_{>0}A0​:R∪rmax​+w→R>0​ and A1:R∪rmin−w→R>0A_1: R \cup r_{min} - w \rightarrow \mathbb{R}_{>0}A1​:R∪rmin​−w→R>0​, one for each token in a pool. CAFs are essential for computing liquidity modifications and swaps.

where a0(r)a_0(r)a0​(r) is the amount of token₀ in rick rrr.

Similarly, for token₁:

A1(ρ)=∑r=rminρa1(r)A_1(\rho) = \sum_{r=r_{min}}^{\rho} a_1(r)A1​(ρ)=∑r=rmin​ρ​a1​(r)

ICAFs are the inverse of CAFs and are crucial for computing swaps in constant time. For each LDF, there are two ICAFs, A0−1(⋅)A_0^{-1}(\cdot)A0−1​(⋅) and A1−1(⋅)A_1^{-1}(\cdot)A1−1​(⋅), defined as:

The computation of ICAFs depends on the specific LDF. For geometric LDFs, ICAFs can be computed via basic arithmetic operations in constant time.

Composing LDFs​

LDFs can be easily composed to create more complex distributions. Given two LDFs LDFaLDF_aLDFa​ and LDFbLDF_bLDFb​, a new LDF LDF′LDF'LDF′ can be created as:

LDF′(x)=w⋅LDFa(x)+(1−w)⋅LDFb(x),w∈[0,1]LDF'(x) = w \cdot LDF_a(x) + (1-w) \cdot LDF_b(x), w \in [0, 1]LDF′(x)=w⋅LDFa​(x)+(1−w)⋅LDFb​(x),w∈[0,1]

This composition property allows for the creation of sophisticated liquidity distributions tailored to specific market conditions or strategies.
